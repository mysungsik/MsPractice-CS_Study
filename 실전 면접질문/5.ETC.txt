1. REDUX 의 설명과 장단점

    Redux 는 상태관리 라이브러리로,
    컴포넌트의 상태 관리 로직을 따로 분리시켜, 효과적으로 관리 할 수 있습니다.
    불필요한 props 전달을 막을 수 있습니다.

    리덕스는 Action 의 타입에 대한 각각의 reducer 함수를 통해, Store 내의 State를 조작합니다.

    리덕스의 단점은, 데이터를 다룰 때마다, 새로운 reducer 함수를 정의해야 하여, 코드량이 증가하는 단점이 있습니다.

2. CORS 에러
    Cross Origin Resource Sharing

    웹서버가, 리소스를 로드할때, "다른 오리진을 통해 로드하지 못하게 하는 HTTP 기반 메커니즘"
    
        * 오리진 : 도메인과 포트번호의 조합

    서버에서 CORS 를 허용해주는 미들웨어를 사용하여, 해결 할 수 있습니다.

3. API 란

    API 또는 응용 프로그래밍 인터페이스는 서로 다른 응용 프로그램 간의 상호 작용 및 통신을 허용하는 기능과 규칙을 제공하는 것

3. REST API 란

    클라이언트와 서버를 독립적으로 사용하는 아키텍처 스타일로
    클라이언트와 서버간의 데이터를 주고 받는 방식입니다.
    HTTP 요청을 통해, CRUD 작업을 실행합니다.

    중요한 특징으로는, 특정 언어와 기술에 구애받지 않는 "유니폼 인터페이스" 를 가집니다.
    즉, IOS, 안드로이드, 웹 등 HTTP 표준을 따른다면, 어디든 사용 가능한 특징을 가집니다.
