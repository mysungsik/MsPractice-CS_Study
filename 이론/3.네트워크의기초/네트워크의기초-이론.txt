1. 네트워크란

    컴퓨터 등의 장치들이 통신기술을 이용하여 구축하는, 연결망
    노드와 링크가 서로 연결되어 있거나, 연결되어 있지 않은 집합체

2. 좋은 네크워크의 구축이란

    "많은 처리량"을 한번에 처리 가능하고, 
    "지연 시간"이 짧고, 
    "장애 빈도가 적으며" 
    "보안성"을 갖춘 네트워크

3. 네트워크 토폴로지

    네트워크 토폴로지 : 노드와 링크가 어떻게 배치되어 있는지에 대한 방식. 또는 연결 형태

    1) 트리 토폴로지
        "계층형 토폴로지" 라고도 부르며 트리 형식을 띈다.
        노드의 추가삭제가 쉽다. 
        특정 트래픽이 집중될때, 하위 노드에 영향을 끼칠 수 있다.

                                [노드]

                    [노드]                  [노드]
                    
                [노드]    [노드]      [노드]      [노드 ]
                ....                    ....

    2) 버스 토폴로지

        중앙 통신 회선 하나에, 여러개의 노드가 연결되어있는 네크워크 구성. 근거리 통신망(LAN)이 대표적인 예이다
        설치비용이 적고, 신뢰성이 우수하다.
        중앙통신회선에서 노드를 추가 삭제하기 쉽다.
        "스푸핑이 가능한 문제점이 있다"

              [노드]            [노드]
                |                 |      
            ㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡ
                        |                   |
                      [노드]              [노드]

        * 스푸핑
            LAN 상에서 "패킷"을 송신과 관련없는 다른 호스트들에게 보내지 않도록 만든 스위치를 마비시켜
            악의적으로, 다른 노드로 패킷을 전송시키는 "데이터 해킹"

    3) 스타 토폴로지

        중앙에 있는 노드에, 모든 노드가 연결된 네트워크 구성
        에러 탐지가 쉽고, 충돌가능성이 낮다.
        장애 노드가 중앙 노드가 아닐 경우, 다른 노드의 영향이 거의 없다
        "중앙 노드에 장애가 발생하면, 모든 노드를 사용할 수 없다. 설치 비용이 고가이다."

                        [노드]  [노드]
                            |    |
                [노드] ---- [노드] ---- [노드]
                            |    |
                        [노드]    [노드] 

    4) 링형 토폴로지

        고리처럼 연결된 망 구성 방식
        에러 탐지가 쉽고, 충동 가능성이 낮다.
        "네크워크 구성 변경이 어렵고, 회선 장애 발생시 전체 네트워크에 영향을 미친다"

        
    5) 메시 토폴로지

        "망형 토폴로지" 라고도 부르며, 그물망처럼 네트워크가 전부 연결되어 있는 구조
        "트래픽 분산처리의 장점"
        "노드구축비용과 운용비용이 고가이다"

4. 토폴로지와 병목현상

    토폴로지가 중요한 이유는 "병목현상을 찾고 제거하는 기준이 되기 때문"

    예시)
        사용자가 서비스를 이용할 때, 지연시간이 긴 문제가 발생했다. (병목현상의 생성)
        관리자는 "대역폭을 늘려" 데이터 처리량을 늘렷으나, 지연시간의 변화가 없었다.

        "관리자"는 "네트워크 토폴로지"를 확인하고, 네트워크의 데이터가 이동하는 경로를 파악하여, "추가적인 회선"을 넣어
        지연시간을 줄일 수 있다.(병목현상의 해결)


5. 네트워크의 분류

    1) LAN (Local Area NetWork)         : 사무실, 건물 크기 정도 크기의 네트워크
    2) MAN (Metropolitan Area NetWork)  : 서울시 등의 도시 규모의 네트워크
    3) WAN (Wide Area NetWork)          : 전 세계적으로 사용 가능한 네트워크

        * 지역이 커질수록 전송속도는 내려가고 혼잡도는 올라간다.

6. 병목현상의 주된 원인

    1) 네트워크 대역폭
    2) 네트워크 토폴로지(네트워크 연결 구조)
    3) 서버 CPU, 메모리 사용량

7. 네트워크 성능 분석 명령어들

    1) ping (Packet INternet Groper)

        대상 노드를 향해, 일정 크기의 패킷을 전송하는 명렁어
        패킷의 수신상태, 도달하는 시간 등을 확인 할 수 있음

        ping 은 "TCP/IP 프로토콜" 을 통해 동작하며, 
        ICMP 프로토콜을 지원하지 않는 기기의 대상으로는 실행불가능
        네트워크 정책상 ICMP 나 traceroute 를 차단하는 경우도 실행 불가능

    2) netstat

        접속되어 있는 서비스들의 "네트워크 상태를 표시"하는데 사용
        주로 서비스이 포트가 열려있는지 확인할때 사용

    3) tracert

        목적 노드까지의 네트워크 경로를 확인할때 사용한다.
        어느 구간에서 응답시간이 느려지는지 확인할때 사용한다.

8. 네트워크 프로토콜 표준화

    네트워크 프토토콜이란, 다른 장치들끼리 "데이터를 주고받기 위해 설정된"
        "공통된 인터페이스"를 말한다.

    IEEE, IETF 라는 표준화 단체에서 정한다.

    IEEE802.3 은 유선 LAN 프로토콜으로, 유선으로 LAN 을 구축할때 쓰이는 프로토콜이고,
    웹에 접속할때 쓰이는 HTTP 라는 프로토콜 을 통해, 노드는 웹서비스를 기반으로 데이터를 주고 받을 수 있는 것이다.


9. TCP/IP 4계층 모델

    "인터넷 프로토콜 스위트" (internet protocol suite) 는 컴퓨터들이 서로 정보를 전달하기 위한 "프로토콜의 집합"
    이를 "TCP/IP 4계층 모델" 혹은 "OSI 7계층 모델" 로 설명한다.
    즉, 계층모델은 "분류별로 나누어 놓은 프로토콜의 집합"인 것이다.

        [ 링, 인, 전, 응]

        "TCP/IP 4계층"
            [응용 계층]
            [전송 계층]
            [인터넷 계층]
            [링크 계층]

    1) 응용 계층

        FTP / HTTP / SSH/ SMTP / DNS 등이 존재
        응용 프로그램(application) 들이 데이터를 교환하기 위해 사용되는 프로토콜이다.
        사용자와 가장 가까우며, "사용자가 응용프로그램과 소통할 수 있게 해주는 프로토콜들의 집합"이다.

    2) 전송 계층

        TCP / UDP / RTP 등 존재
        통신 노드간의 연결을 제어하고, 자료의 송수신을 담당한다. = 통신 서비스 제공
        애플리케이션 계층과 인터넷 계층 사이의 데이터가 전달될 때의 중계 역할을 한다.

    3) 인터넷 계층

        IP / ARP/ ICMP 등이 존재
        장치로 부터 받은 "네트워크 패킷"을, IP 주소로 지정된 "목적지로 전송 하기위해 사용되는 계층"
        상대방이 제대로 받았는지에 대해, 보장하지 않는, "비연결형적인 특징"이 있다.

    4) 링크 계층

        전선, 광섬유 무선 등 실질적인 데이터를 전달하며, 장치간의 신호를 주고받는 "규칙"을 정하는 계층
        "접근계층" 이라고도 한다.

10. 계층간의 송수신 과정

    [예시]
        컴퓨터로 HTTP 프로토콜을 이용하여, 웹 서버의 데이터를 요청한다.

                클라이언트      => 애플리케이션계층 => 전송 계층 => 인터넷 계층 => 링크 계층     [캡슐화 과정]
            => 서버측 링크 계층 => 서버측 인터넷 계층 => 서버측 전송 계층 => 서버측 애플리케이션 계층  [비캡슐화 과정]

        "캡슐화 과정"을 통해, 상위계층의 헤더와 데이터를, 하위 계층의 데이터에 포함시키는 과정 을 통해, 압축
        서버에 도착하면 "비캡슐화 과정"을 통해, 하위계층에서 상위계층으로 가며, 각 부분의 헤더를 삭제, 데이터화 되어 PDU인 "메시지"로 전달된다.

        *PDU (protocol Data Unit)
            : 계층에서 계층으로 데이터가 전달 될 때, 데이터의 한 덩어리 단위

        *각 계층별 PDU 를 부르는 명칭
            1) 애플리케이션계층 : 메시지
            2) 전송 계층 : 세그먼트(TCP), 데이터그램(UDP)
            3) 인터넷 계층 : 패킷
            4) 링크 계층 : 프레임(데이터링크), 비트(물리 계층)

11. OSI 7계층 모델

    "TCP/IP 4계층"에서 "애플리케이션 계층"을, "세션, 표현, 응용 계층"으로 분리하며
                        "링크 계층" 을, "데이터 링크 계층, 물리 계층" 으로 분리하고,
                        "인터넷 계층" 을 "네트워크 계층" 으로 부른다. 
                        는 차이가 있다.
